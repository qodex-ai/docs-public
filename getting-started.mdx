## **Step 1: Import or Discover Your APIs**

### **Option 1: Upload Swagger or Postman Collection**

- Go to **AI Agent**
- Click **Add a Collection** dropdown ‚Üí \*\*Import Collection \*\*‚Üí **Postman** or **Swagger**
- Upload:
  - swagger.json / swagger.yaml
  - .postman_collection.json

![Add Collection Gi](/images/Add-Collection.gif)

### **Option 2: Use Qodex SDK for API Discovery**

- Run this command from your backend:

  ```bash
  curl -sSL https://raw.githubusercontent.com/qodex-ai/swagger-bot/refs/heads/main/run.sh -o script.sh && chmod +x script.sh && ./script.sh --project-api-key YOUR_PROJECT_KEY
  ```

This generates a live Swagger spec from your running server and auto-imports it into Qodex.

- Works with Node.js, Rails, Python, etc.
- Auto-discovers all APIs \+ methods

## **Step 3: Request Data & Test Rules**

### **Generating Request Data:**

- In each test scenario:

  Click **Create Request Data** (top-right)

  ‚Üí Qodex generates:
  - Method
  - Path
  - Headers / Auth
  - Params
  - Request Body

If AI needs more info, it will prompt you during setup.

### **Adding Test Rules:**

- Scroll to **Test Rules** inside each test case
- You can:
  - Select from **Suggested** rules
  - Write custom rules in **Plain English** (converted to JS)

**Examples:**

- _‚ÄúExpect status code 200‚Äù_
- _‚ÄúVerify token is not expired‚Äù_
- _‚ÄúValidate response contains user_id‚Äù_

## **Step 5: Running Tests**

### **Option A: In-App**

- Go to **Test Scenarios** or **Test Suites**
- Click **Run**
- Select environment
- View results in **Build Tab** or **Build Reports**

### **Option B: CLI**

```bash
qodex test run --env=.env.staging
```

### **Option C: CI/CD**

Supports GitHub Actions, Jenkins, CircleCI:

```yaml
- name: Run Qodex Tests
  run: qodex test run --env=.env.prod
```
