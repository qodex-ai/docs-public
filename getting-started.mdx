Welcome to [**Qodex.ai**](http://Qodex.ai) — your AI-powered platform to **discover, test, and secure APIs** with zero manual scripting.

This guide will help you quickly go from signup to running your first API tests — whether you’re onboarding a new microservice or automating coverage across hundreds of endpoints.

## What is Qodex.ai?

[Qodex.ai](http://Qodex.ai) is an AI-first platform designed to make API testing effortless:

- Discover APIs — no OpenAPI spec required
- Auto-generate functional, regression & security tests in plain English
- Run tests manually, on schedule, or in CI/CD
- Auto-heal tests as APIs change
- Sync test code to GitHub
- Get real-time alerts for failures, security risks & regressions

> No code. No manual test writing. Just describe what you want — Qodex handles the rest.

## Quickstart Steps

1. **Sign up**

   Go to [app.qodex.ai](http://app.qodex.ai) → Create your account.

   You will land on the **AI Agent page** after login: /ai-agent.
2. **Import or Discover your APIs**

   In the AI Agent:

   → **Add Collection** dropdown (top-left) → Upload:
   - Swagger (.json or .yaml)
   - Postman collection (.json)

   OR:

   Run Qodex’s SDK to auto-discover live APIs:

   ```bash
   curl -sSL https://raw.githubusercontent.com/qodex-ai/swagger-bot/refs/heads/main/run.sh -o script.sh && chmod +x script.sh && ./script.sh --project-api-key YOUR_PROJECT_KEY
   ```

   Once imported, your API endpoints will appear in the platform — ready for test generation.
3. **Generate your First Test**

   From the AI Agent chat:

   Type something like:

   _“Test login flow with valid \+ invalid credentials”_

   or

   _“Run OWASP Top 10 tests on my APIs”_

   AI will generate test scenarios → you can review and **Save**.

   Tests will now appear under **Test Scenarios** tab.
4. **Run Tests**
   - Go to **Test Scenarios** or **Test Suites**
   - Select active test(s)
   - Click **Run**
   - Choose environment (default, staging, prod)
   - View results in **Build Tab** or **Build Reports**
5. **Sync with GitHub (Optional but Recommended)**

   In **Test Scenarios** page:
   - Click **Sync to GitHub**
   - Test code is pushed to /tests/qodex/
   - Now you can run locally or in CI/CD
6. **Set up Notifications**
   - Add active Test Scenarios into a Suite
   - Create a \*\*Test Plan \*\*→ Add suite(s) to a Test Plan
   - Add Slack / Email / Webhook alerts from the details tab in the right
   - Stay informed when tests fail or APIs drift

That’s it, its that easy\!

You’re now ready to start testing your APIs.
